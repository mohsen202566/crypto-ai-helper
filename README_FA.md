# ربات تطبیقی کریپتو — نسخه چندتایم‌فریمی OKX / Bybit / Binance / Toobit

این پروژه برای Python 3.10 به بالا ساخته شده و سه سطح سیگنال **Initial، Medium و Real** را به‌صورت مرحله‌ای و یادگیرنده مدیریت می‌کند. داده اصلی از OKX دریافت می‌شود؛ Bybit و Binance Futures دو جایگزین مستقل هستند و Toobit فقط برای اجرای واقعی و خواندن نتیجه مالی استفاده می‌شود. هر چرخه تحلیلی کاملاً از یک صرافی ساخته می‌شود و داده صرافی‌ها با هم مخلوط نمی‌شود.

> تست‌های پروژه هیچ سفارش واقعی ارسال نمی‌کنند. ترید واقعی بعد از هر استارت اجباری خاموش است و فقط با دستور `ترید فعال` روشن می‌شود.

## زنجیره مقاوم داده بازار

ترتیب پیش‌فرض منابع:

```text
OKX → Bybit → Binance Futures
```

اگر یک منبع Rate Limit (`429`)، خطای دسترسی، Timeout یا قطع شبکه بدهد، برای مدت مشخص وارد Cooldown می‌شود. در این مدت همه ارزها مستقیماً از منبع سالم بعدی خوانده می‌شوند و همان خطا را برای تک‌تک ارزها تکرار نمی‌کنند. تعداد درخواست هم‌زمان OKX و فاصله شروع درخواست‌ها محدود شده است. ترتیب و محدودیت‌ها از `.env` قابل تنظیم‌اند.

## معماری تایم‌فریم

تایم‌های معامله:

- `30m`
- `1H`
- `4H`
- `1D`

تایم معامله مسئول **جهت، قدرت، ساختار و هندسه TP/SL** است. تایم پایین‌تر فقط زمان ورود را دقیق می‌کند:

| تایم معامله | تایم‌های مجاز ورود | ورود پیش‌فرض |
|---|---|---|
| 30m | 15m، 5m | 15m |
| 1H | 15m، 5m، 30m | 15m |
| 4H | 1H، 30m | 1H |
| 1D | 4H، 1H | 4H |

روی پیام هر سیگنال، تایم معامله و تایم ورود جداگانه نمایش داده می‌شود.

## RR و TP/SL

RR عدد ثابت سراسری نیست. هر ترکیب `ارز + جهت + تایم معامله` پروفایل و یادگیری مستقل دارد. محدوده امن اولیه:

| تایم معامله | RR پیش‌فرض | محدوده مجاز |
|---|---:|---:|
| 30m | 2.0 | 1.5 تا 2.5 |
| 1H | 2.2 | 1.7 تا 3.0 |
| 4H | 2.8 | 2.0 تا 4.0 |
| 1D | 3.5 | 2.2 تا 5.0 |

ابتدا Stop پشت نقطه ابطال ساختاری و بیرون نویز همان تایم قرار می‌گیرد. سپس TP با ATR، ساختار، قدرت، رفتار بازار، هزینه‌ها و RR یادگرفته‌شده تعیین می‌شود. موتور می‌تواند RR، ضرایب ATR، نوع ورود و تایم ورود را فقط از مسیر سناریوی تک‌تغییری و تأیید مرحله‌ای اصلاح کند.

## مسیر یادگیری

```text
سناریوی تک‌تغییری
        ↓
Initial
        ↓
Medium
        ↓
Real
```

آمار، سناریوها، Champion/Challenger، ارتقا، Rollback و تشخیص Stop برای هر `ارز + جهت + تایم معامله` جدا ذخیره می‌شوند. یک موفقیت یا شکست به‌تنهایی تنظیمات اصلی را عوض نمی‌کند.

## قفل‌ها و اجرای واقعی

- در هر `ارز + تایم معامله` فقط یک سیگنال رسمی باز می‌تواند وجود داشته باشد؛ جهت یا سطح دیگر همان تایم تا بسته‌شدن آن قفل است.
- سیگنال‌های مجازی یک ارز روی تایم‌های متفاوت می‌توانند هم‌زمان مانیتور شوند.
- برای ایمنی، در Toobit هم‌زمان فقط یک پوزیشن واقعی برای هر ارز مجاز است؛ حتی اگر چند تایم سیگنال آماده Real داشته باشند.
- Initial و Medium محدودیت کلی اسلات ندارند.
- Medium با مارجین، لوریج، کارمزد، اسلیپیج و Funding شبیه‌سازی می‌شود.
- حداقل سود خالص Medium/Real برابر ۰٫۰۵ USDT است.
- دو Stop واقعی متوالی، همان `ارز + تایم معامله` را به Medium Relearn می‌برد تا دوباره اثبات شود.

## ساختار فایل‌ها

- `main.py`: چرخه عمر و Workerها
- `config.py`: تایم‌ها، RRها، پیش‌فرض‌ها و بازه‌ها
- `models.py`: مدل‌های داده چندتایمی
- `storage.py`: دو دیتابیس، WAL، Backup و Migration خودکار
- `symbol_registry.py`: اعتبارسنجی Toobit و حداقل دو منبع عمومی از OKX/Bybit/Binance و انتخاب ۳۵ ارز فعال
- `market_data.py`: OKX اصلی، Bybit و Binance fallback، Circuit Breaker و محدودکننده درخواست
- `feature_engine.py`: ابزارها و پروفایل مستقل هر تایم
- `behavior_engine.py`: تشخیص احتمالاتی رفتار و کیفیت ورود
- `tp_sl_engine.py`: TP/SL و RR تطبیقی هر تایم
- `signal_engine.py`: اسکن چهار تایم و صدور Initial/Medium/Real
- `scenario_lab.py`: سناریوهای تک‌تغییری
- `learning_engine.py`: ثبت نتایج و یادگیری مرحله‌ای
- `validator.py`: Promotion، Challenger/Champion و Rollback
- `virtual_monitor.py`: مانیتور سیگنال‌های مجازی
- `toobit_client.py`: API فیوچرز Toobit
- `trade_engine.py`: ارسال سفارش واقعی بدون انتظار بلاک‌کننده
- `real_monitor.py`: تأیید بازشدن و خواندن نتیجه واقعی
- `telegram_ui.py`: پیام سیگنال، نتیجه و پنل‌ها
- `self_test.py`: تست آفلاین یکپارچگی

## رفتار شروع

1. ترید واقعی خاموش می‌شود.
2. ۱۰۰ قرارداد Toobit که روی حداقل دو منبع عمومی از OKX، Bybit و Binance Futures موجود باشند اعتبارسنجی می‌شوند.
3. ۳۵ ارز فعال و ۶۵ ارز ذخیره انتخاب می‌شوند.
4. برای هر ارز فعال، هر دو جهت و هر چهار تایم معامله پروفایل مستقل ساخته یا بازیابی می‌شود.
5. هر پروفایل از کندل‌های همان تایم ساخته می‌شود؛ داده 1D از 5m یا 15m شبیه‌سازی نمی‌شود.
6. تا آماده‌شدن همه پروفایل‌های فعال، صدور سیگنال بسته می‌ماند.
7. Initial و Medium شروع می‌شوند؛ Real همچنان خاموش است.

در Restart یا Git pull، دیتابیس، Stage، Champion، نتایج و تاریخچه یادگیری حفظ می‌شوند. Migration نسخه قدیمی به ساختار چندتایمی خودکار است.

## دیتابیس‌ها

### `runtime.db`

تنظیمات، سیگنال‌ها، قفل `ارز + تایم`، اسلات، پوزیشن واقعی، Snapshot حساب، Offset تلگرام و سلامت Workerها.

### `learning.db`

پروفایل‌های مستقل تایم‌ها، نتایج، سناریوها، نسخه‌ها، Champion/Challenger، تشخیص Stop، مسیر بعد از نتیجه و ممیزی یادگیری.

هر دو دیتابیس SQLite WAL هستند و فایل‌های DB در Git قرار نمی‌گیرند.

## نصب روی VPS

```bash
cd /root
git clone <YOUR_REPOSITORY_URL> forex-signal-bot
cd /root/forex-signal-bot
sudo ./install_service.sh
sudo nano /etc/forex-signal-bot.env
sudo systemctl start forex-signal-bot.service
journalctl -u forex-signal-bot.service -n 100 -f
```

Python باید حداقل 3.10 باشد. کلید Toobit به Futures Trading و خواندن حساب/پوزیشن نیاز دارد؛ دسترسی برداشت لازم نیست و بهتر است خاموش باشد.

## نکات عملیاتی

- خرابی یک نماد باعث Crash کل ربات نمی‌شود؛ نماد Skip/Cooldown می‌شود و اسکن ادامه دارد.
- کش کندل به تفکیک `source + symbol + timeframe` نگهداری می‌شود. در چرخه بعد فقط دنباله کوتاه همان تایم از همان منبع تازه می‌شود.
- OKX، Bybit و Binance در یک Bundle با هم مخلوط نمی‌شوند؛ کل تحلیل هر چرخه فقط از یک صرافی است.
- خطای 429، قطع شبکه یا 403 یک صرافی، Circuit Breaker همان منبع را فعال می‌کند و ربات بدون تکرار طوفانی درخواست‌ها به منبع بعدی می‌رود.
- مقدار خالی تایم‌فریم پیش از ارسال درخواست اصلاح می‌شود؛ بنابراین درخواست `bar=` خالی به OKX ارسال نمی‌شود.
- مهلت ۷۰ثانیه‌ای تأیید پوزیشن از شروع واقعی Submit محاسبه می‌شود و بلاک‌کننده نیست.
- پوزیشن دستی یا خارج از ربات که Toobit گزارش کند نیز اسلات واقعی مصرف می‌کند.
- Feed خبر اختیاری است و بدون `NEWS_CALENDAR_URL` هیچ خبر ساختگی تولید نمی‌شود.
